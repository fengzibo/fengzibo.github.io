作用域链及闭包
js
js 2018-12-29

前言

在之前的执行上下文中，我们知道了每个函数都有一个执行上下文，对于每个执行上下文都有三个重要属性：

  • 作用域链
  • 变量对象
  • this 指向

作用域链

作用域链，是由当前环境与上层环境的一系列变量对象组成，它保证了当前执行环境对符合访问权限的变量和函数的有序访问。

让我们来结合一个例子，更好的理解作用域链到底是什么

var global = 0;
function one(){
    var one = 1
    two()
    function two(){
        var two = 2
        three()
        function three(){
            var three = 3
            console.log(global + one + two +three)
        }
    }
}
one() //6

很明显，在这例子中在全局环境中调用了 one() ,然后 one 又调用了 two ，two 有调用了 three，产生了4个执行上下文，那么自然就有4个变量对象，可以用伪代码来表述下：

GlobalExecutionContext = {
    scopeChain:[[Glbal Vo]],
    variableObject:{
        global: 0,
        ...
    },
    this:{...}
}

OneExecutionContext = {
    scopeChain:[[One Vo],[Glbal Vo]],
    variableObject:{
        one: 1,
        ...
    }, 
    this:{...}
}

TwoExecutionContext = {
    scopeChain:[[Two Vo],[One Vo],[Glbal Vo]],
    variableObject:{
        two: 2,
        ...
    }, 
    this:{...}
}

ThreeExecutionContext = {
    scopeChain:[[Three Vo],[Two Vo],[One Vo],[Glbal Vo]],
    variableObject:{
        three: 3,
        ...
    }, 
    this:{...}
}

上面伪代码中的 scopeChain 就是作用域链 [xxx Vo] 指的是对应执行上下文执行阶段的变量对象，作用域链添加规则类似于 scopeChain = [自身的VO].concat([父级的scopeChain])，所以当代码执行到 console.log 的时候，首先通过查看作用域链并检查第一个变量对象来解析 [Three Vo]。它检查 global 是否存在于内部，[Three Vo] 但找不到具有该名称的任何属性，因此继续检查下一个 [Vo]。

这样按照顺序检查每个变量名是否存在，如果没找到就 抛出 Uncaught ReferenceError 的错误。因此，上面打印结果为6。

知道了作用域链是怎么回事，我们就可以解释这个例子了

var arr = []
for(var i=0; i<5; i++){
    arr.push(
        function inner(){
            console.log(i)
        }
    )
}
arr[0]()
arr[1]()
arr[2]()
arr[3]()
arr[4]()

乍一看，以为是0，1，2，3，4,知道了作用域链我们就可以知道答案都是5。我们可以用伪代码实力分析一波
当执行arr[0]()的时候，for 循环已经结束，所以 i 是5

GlobalExecutionContext = {
    scopeChain:[[Glbal Vo]],
    variableObject:{
        arr: [
            0:function inner(){
                console.log(i)
            },
            1:...
            ...
        ],//有5个
        i:5
        ...
    },
    this:{...}
}
arr[0]ExecutionContext={
    scopeChain:[[arr[0] Vo],[Glbal Vo]],
    variableObject:{...},
     this:{...}
}

arr[0]的i是在[Glbal Vo]中找到的，所以结果是5，

那如何能让它达到预期，返回的是0，1，2，3，4呢？我们只需要在 [Glbal Vo] 中 i 的值变化之前调用函数就好了

var arr = []
for(var i=0; i<5; i++){
    arr.push(
        function inner(){
            console.log(i)
        }
    )
    arr[i]()
}

或者利用闭包的形式

var arr = []
for(var i=0; i<5; i++){
    arr.push(
        (function inner(i){
            return function(){
                console.log(i)
            }
        })(i)
    )
}
arr[0]()
arr[1]()
arr[2]()
arr[3]()
arr[4]()

闭包

闭包的概念好难写，个人的是这么理解的

一个函数 A 在其内部创建了一个函数 B 并返回，即使这个创建它的函数 A 的执行上下文已经销毁，函数 B 的引用仍然存在,并且函数 B 访问了函数 A 中变量对象的值，那么就产生了闭包

我们就拿上面的例子，我们详细分析一波。

在循环中一个自执行函数 inner 调用，创建执行上下文，内部创建了一个匿名函数并返回 push 到数组中，并且匿名函数访问了 inner 的传参，inner 执行完毕销毁，匿名函数仍然存在数组中。还是拿伪代码更有助于理解

globalContext = {
    variableObject: {
        i: (根据循环从0-5)
        arr:[...]
        ...
    }
}
//根据循环 innerContext 进出5次 
innerContext = {
    scopeChain: [[inner Vo],[Glbal Vo]],
    variableObject: {
        arguments: {
            0: (根据循环从0-4),
            length: 1
        },
        i: (根据循环从0-4)
        ...
    }
}

开始调用 arr[0]()，执行阶段

globalContext = {
    variableObject: {
        i: 5
        arr:[...]
        ...
    }
}

arr[0]Context = {
    scopeChain:[[arr[0] Vo],[inner[0] Vo],[Glbal Vo]]
    variableObject:{...}
}

按照常理来讲，inner 执行完毕后销毁，但由于 arr 引用了 inner创建的匿名函数，就导致 inner 的 变量对象（[inner[i] Vo]）被保留了下来,[arr[i] Vo]中找不到的变量都在 [inner[i] Vo] 找到了，所以结果返回了0，1，2，3，4。

使用闭包的注意点

  • 由于闭包会使得函数中的变量对象都被保存在内存中，内存消耗很大，所以不能滥用闭包，否则会造成网页的性能问题，在IE中可能导致内存泄露。解决方法是，在退出函数之前，将不使用的局部变量全部删除。
  • 闭包会在父函数外部，改变父函数内部变量的值。所以，如果你把父函数当作对象（object）使用，把闭包当作它的公用方法（Public Method），把内部变量当作它的私有属性（private value），这时一定要小心，不要随便改变父函数内部变量的值。

补充

突然想到原型链，当我们尝试访问某个对象的属性，我们会先从作用域链中寻找，找不到再从原型中寻找，再找不到就返回 undefind 或报错
