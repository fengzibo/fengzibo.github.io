js继承方式
js
js 2018-12-24

前言

本文讲讲JavaScript各种继承方式和优缺点。并加以验证

1.原型链继承

将父类的实例作为子类的原型

function Father(){
    this.surname = '吴'
    this.property = ['house','car']
}
Father.prototype.getSurname = function () {
    console.log(this.surname)
}

function Children(){

}

Children.prototype = new Father();
var son = new Children();
var daughter = new Children();
son.getSurname() //吴
console.log(son.property) //['house','car']
son.getSurname() //吴
console.log(daughter.property) //['house','car']


son.__proto__.surname = '王'
son.property.splice(1,1,'newCar')
son.getSurname() //王
console.log(son.property) //['house','newCar']
daughter.getSurname() //王
console.log(daughter.property) //['house','newCar']

console.log(son instanceof Father); //true 
console.log(son instanceof Children); //true

优点：

  • 非常纯粹的继承关系，实例是子类的实例，也是父类的实例
  • 父类新增原型方法/原型属性，子类都能访问到

缺点：

  • 来自原型对象的所有属性被所有实例共享
  • 创建子类实例时，无法向父类构造函数传参

2.借用构造函数(经典继承)

使用父类的构造函数来增强子类实例，等于是复制父类的实例属性给子类（没用到原型）

function Father(name){
    this.surname = '吴'
    this.property = ['house','car']
    this.childName = name
}
Father.prototype.getSurname = function () {
    console.log(this.surname)
}

function Children(name){
    Father.call(this,name)
}
var son = new Children('darkbo')
son.property.splice(1,1,'newCar')
var daughter = new Children('sandy')
console.log(son.property) //["house", "newCar"]
console.log(daughter.property) //["house", "car"]
console.log(son instanceof Father); //false 
console.log(son instanceof Children); //true
son.getSurname() //Uncaught TypeError: son.getSurname is not a function

优点：

  • 避免了共享属性
  • 创建子类实例时，可以向父类传递参数

缺点：

  • 实例并不是父类的实例，只是子类的实例
  • 只能继承父类的实例属性和方法，不能继承原型属性/方法
  • 无法实现函数复用，每个子类都有父类实例函数的副本，影响性能

3.组合继承

组合 原型链继承 和 借用构造函数继承。通过调用父类构造，继承父类的属性并保留传参的优点，然后通过将父类实例作为子类原型，实现函数复用

function Father(name){
    this.surname = '吴'
    this.property = ['house','car']
    this.childName = name
}
Father.prototype.getSurname = function () {
    console.log(this.surname)
}
Father.prototype.getAge = function () {
    console.log(this.age)
}

function Children(name,age){
    Father.call(this,name)
    this.age = age
}
Children.prototype = new Father()
Children.prototype.constructor = Children;


var son = new Children('darkbo',18)
son.property.splice(1,1,'newCar')
var daughter = new Children('sandy',5)
console.log(son.property) //["house", "newCar"]
console.log(daughter.property) //["house", "car"]
console.log(son instanceof Father); //true 
console.log(son instanceof Children); //true
son.getSurname() //吴
son.getAge() //18
daughter.getAge() //5

优点：

  • 融合原型链继承和构造函数的优点，是 JavaScript 中最常用的继承模式

缺点：

  • 调用了两次父类构造函数，多消耗了一点内存

4.原型式继承

function create(o){
    function F(){}
    F.prototype = o
    return new F()
}

let father = {
    property : ['house','car']
}

let son = create(father)

son.property.splice(1,1,'newCar')

let daughter = create(father)

console.log(son.property) //["house", "newCar"]
console.log(daughter.property) //["house", "newCar"]

同样共享了属性和方法

5.寄生式继承

创建一个仅用于封装继承过程的函数，该函数在内部以某种形式来做增强对象，最后返回对象

function createObj (o) {
  var clone = Object.create(o)
  clone.sayName = function () {
    console.log('hi')
  }
  return clone
}

缺点：跟借用构造函数模式一样，每次创建对象都会创建一遍方法。

6.寄生组合继承

通过寄生方式，砍掉父类的实例属性，这样，在调用两次父类的构造的时候，就不会初始化两次实例方法/属性，避免的组合继承的缺点

function Father(name){
    this.surname = '吴'
    this.property = ['house','car']
    this.childName = name
}
Father.prototype.getSurname = function () {
    console.log(this.surname)
}
Father.prototype.getAge = function () {
    console.log(this.age)
}

function Children(name,age){
    Father.call(this,name)
    this.age = age
}


var f = function(){}
f.prototype = Father.prototype
Children.prototype = new f()
// Children.prototype = new Father()
Children.prototype.constructor = Children;


var son = new Children('darkbo',18)
son.property.splice(1,1,'newCar')
var daughter = new Children('sandy',5)
console.log(son.property) //["house", "newCar"]
console.log(daughter.property) //["house", "car"]
console.log(son instanceof Father); //true 
console.log(son instanceof Children); //true
son.getSurname() //吴
son.getAge() //18
daughter.getAge() //5

封装一下：

function create(o){
    function F(){}
    F.prototype = o
    return new F()
}
function Prototype(son,father){
    let prototype = create(father.prototype)
    prototype.constructor = son
    son.prototype = prototype
}
function Father(name){
    this.surname = '吴'
    this.property = ['house','car']
    this.childName = name
}
Father.prototype.getSurname = function () {
    console.log(this.surname)
}
Father.prototype.getAge = function () {
    console.log(this.age)
}

function Children(name,age){
    Father.call(this,name)
    this.age = age
}
Prototype(Children,Father)
let son = new Children('darkb',18)
son.getAge()
son.property.splice(1,1,'newCar')
let daughter = new Children('sandy',5)
daughter.getAge()
console.log(son.property) //["house", "newCar"]
console.log(daughter.property) //["house", "car"]
console.log(son instanceof Father); //true 
console.log(son instanceof Children); //true

这种方式的高效率体现它只调用了一次 Father 构造函数，并且因此避免了在 Father.prototype 上面创建不必要的、多余的属性。与此同时，原型链还能保持不变；因此，还能够正常使用 instanceof 和 isPrototypeOf。开发人员普遍认为寄生组合式继承是引用类型最理想的继承范式。 ————————引用自《JavaScript 高级程序设计》

7.ES6 extends

//父类
class Father{
    constructor(){
        this.surname = '吴'
        this.property = ['house','car']
    }
    getAge(){
        console.log(this.age)
    }
}

//子类
class Children extends Father{
    constructor(name,age){
        super()
        this.name = name
        this.age = age
    }
}
let son = new Children('darkbo',18)
son.getAge() //18
son.property.splice(1,1,'newCar')
console.log(son.property)
let daughter = new Children('sandy',5)
console.log(daughter.property) //["house", "car"]
console.log(son instanceof Father); //true 
console.log(son instanceof Children); //true

优点：不用手动设置原型。

缺点：新语法，只要部分浏览器支持，需要转为 ES5 代码。
