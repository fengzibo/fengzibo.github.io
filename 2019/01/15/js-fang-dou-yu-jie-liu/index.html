js防抖与节流
js
js 2019-01-15

前言

在一个项目中，用户进行一些频繁的操作，如

  • 用户输入账号密码，我们前端校验或向后台请求校验的时候
  • 用户频繁的调整浏览器窗口大小
  • 用户快速滚动滚动条
  • 用户鼠标频繁的移入移出点击等等

这些频繁执行 DOM 操作、资源加载等重行为，可能会导致 UI 停顿甚至浏览器崩溃。我们要去控制这些行为，防止不良的用户体验出现，就要用到我们的防抖和节流了

我们来看个用户输入的例子
写一个 index.html 文件

<!DOCTYPE HTML>
<html lang="zh-CN">
<head>
    <meta charset="utf-8">
</head>

<body>
    <input type="text" id="search" />
    <script src="debounce.js"></script>
</body>
</html>

debounce.js 文件如下

var input = document.getElementById('search')
function verify(){
    console.log(111)
}
input.addEventListener('keyup', function(e) {
  verify()
})

看看效果：

输入了这些字符，短短的时间内执行了这么多次，如果是 ajax 请求将会向后台请求好多次，会加重后台压力，并且加载是有时间的，如果代码复杂些，会造成不可预知的 bug。

要解决这些问题，有两种方式 debounce 防抖和 throttle 节流，我们先来看看防抖

防抖

防抖简单来说就是当调用函数 n 秒后，才会执行该函数，若在这 n 秒内又调用该函数则将重新计算执行时间。

来个简单实现

function debounce(fn, wait) {
    var t;
    return function () {
        clearTimeout(t)
        t = setTimeout(fn, wait);
    }
}

在原来的基础上这么调用：

input.addEventListener('keyup', debounce(verify,1000))

来看看效果

我们可以看到触发 verify 函数的频率降低了，随你怎么输入，等你输入停止后 1000ms 才执行事件，但是如果我们这里有个问题，this 和 event 获取不到了，这里就不写例子验证，有疑问的同学自己撸遍代码，我们来优化下

function debounce(fn, wait) {
    var t;
    return function () {
        var self = this //获取this指向
        var args = arguments //获取参数
        clearTimeout(t)
        t = setTimeout(function(){
            fn.apply(self,args)
        }, wait);
    }
}

//改写verify
function verify(e){
    console.log(this.value)
    console.log(e)
}

代码到这差不多完善了，但是我不想事件停止触发后才执行，我希望我输入第一个字符直接触发，期间怎么输入都不触发事件，直到停止触发n秒后，才可以重新触发执行。那我们来实现一下

function debounce(fn, wait,immediate = false) { //加个参数进行控制是否立即执行,初始值为false
    var t;
    return function () {
        var self = this //获取this指向
        var args = arguments //获取参数
        if(t) clearTimeout(t)
        if(immediate){
            var callNow = !t 
            t = setTimeout(function (){
                t = null
            }, wait);
            if(callNow){
                fn.apply(self,args)
            }
        }else{
            t = setTimeout(function(){
                fn.apply(self,args)
            }, wait);
        }
    }
}

//调用方式
input.addEventListener('keyup', debounce(verify,1000,true))

同样的来看效果

我们可以看到输入之后立即执行，停止输入等待时间大于1000后，再次输入又立即执行。

关于防抖我们在考虑一个小需求，当输入时间间隔是 5000ms，immediate 为 true 的时候，我只能等到 5000ms 后才能重新触发事件，现在我希望有一个按钮，点击之后，又可以重新输入并立即执行，为此，我们优化一下

function debounce(fn,wait,immediate){
    var t
    var debounce = function () {
        var self = this,
            args = arguments
        if(t) clearTimeout(t)
        if(immediate){
            var callNow = !t
            t = setTimeout(() => {
                t = null
            }, wait);
            if(callNow) fn.apply(self,args)
        }else{
            t = setTimeout(() => {
                fn.apply(self,args)
            }, wait);
        }
    }
    debounce.cancel = function () {
        clearTimeout(t)
        t = null
    }
    return debounce
}

如何调用

var fn = debounce(verify,5000,true)
input.addEventListener('keyup', fn)
document.getElementById('button').addEventListener('click',fn.cancel)

效果如下

至此，防抖就基本实现了

节流

节流简单的来说就是对于连续的函数调用，每间隔一段时间，只让其执行一次。
初步实现方式有两种，一种是使用时间戳，一种是设置定时器。

使用时间戳

设置一个对比时间戳，当触发事件时，使用当前时间戳减去对比时间戳，如果差值大于设定的间隔时间，则执行函数，并用当前时间戳替换对比时间戳；如果差值小于设定的间隔时间，则不执行函数。

function throttle(fn,wait){
    var previous = 0
    return function () {
        var now = new Date().getTime(),
            self = this,
            args = arguments

        if(now - previous > wait){
            fn.apply(self,args)
            previous = now
        }
    }
}

调用方式

input.addEventListener('keyup', throttle(verify,1000))

效果展示

当我们对输入框快速的输入，事件会立即执行，然后每过一秒执行一次，假如我们在11：19：16之后输入未到17秒之前停止，将不会再执行事件，这就是所谓的有头无尾

使用定时器

当首次触发事件时，设置定时器，wait 毫秒后执行函数并将定时器置为 null，之后触发事件时，如果定时器存在则不执行，如果定时器不存在则再次设置定时器。

function throttle(fn,wait){
    var t,
        previous = 0

    return function(){
        var self = this,
            args = arguments

        if(!t){
            t = setTimeout(function(){
                t = null
                fn.apply(self,args)
            },wait)
        }   
    }
}

效果展示

当我们对输入框快速的输入，事件不会立即执行，输入 1000ms 后才初次执行，然后每 1000ms 执行一次，当在11：29：17之后未到18秒之前停止输入的的时候，最后仍然会执行一次，这就是所谓的有无头有尾

那我们想要个有头有尾的，那就结合两者的优势，双剑合璧

结合版

function throttle(fn,wait){
    var t,
        previous = 0

    return function(){
        var now = new Date().getTime(),
            self = this,
            args = arguments,
            remaining = wait - (now - previous) //距离下次函数执行的剩余时间

        if(remaining <= 0 || remaining > wait){ // 如果无剩余时间或系统时间被修改
            if(t){
                clearTimeout(t)
                t = null
            }
            previous = now
            fn.apply(self,args)
        }else if(!t){
            t = setTimeout(function(){
                previous = new Date().getTime()
                t = null
                fn.apply(self,args)
            }, remaining);
        }
    }
}

效果展示

我们来看下代码是如何执行的

  • 首次打印：remaining 为负数，立即执行函数并更新 previous。
  • 第二次打印：remaining 为正数，!t 为 true 设置定时器，remaining 秒后执行函数并更新 previous，将定时器设为 null
  • 之后的打印：
    • 若定时器内函数执行，remaining 为正数，则继续设置定时器，remaining 秒后执行函数
    • 若定时器内函数未执行，但 remaining 小于等于0，清除定时器并置为 null，立即执行函数，更新时间戳，下一次触发继续设定定时器

有头有尾实现成功，但是有时候想要上面两种情况，难道我要写三个函数，太多余了，再优化下

function throttle(fn,wait,options){
    var t,
        previous = 0

    if(!options) options = {}
    return function(){
        var self = this,
            args = arguments,
            now = new Date().getTime()
        if(!previous && options.leading === false) previous = now
        var remaining = wait - (now - previous) //距离下次函数执行的剩余时间

        if(remaining <= 0 || remaining > wait){
            if(t){
                clearTimeout(t)
                t = null
            }
            previous = now
            fn.apply(self,args)
            // 解除引用，防止内存泄漏
            if (!t) self = args = null
        }else if(!t && options.trailing !== false){
            t = setTimeout(function(){
                previous = options.leading === false ? 0 : new Date().getTime()
                t = null
                fn.apply(self,args) 
                // 解除引用，防止内存泄漏
                if (!t) self = args = null
            }, remaining);
        }
    }
}

注意这样的实现方式有个问题，leading：false 和 trailing: false 不能同时设置，因为 trailing 设置为 false，停止触发的时候不会设置定时器，所以只要再过了设置的时间，再输入的话，就会立刻执行，就违反了 leading: false。这种无头无尾的需求挺小众的，暂时不深入讨论

同样的，我们也给 throttle 添加 cancel 方法

function throttle(fn,wait,options){
    var t,
        pervious = 0

    if(!options) options = {}
    var throttle = function(){
        var self = this,
            args = arguments,
            now = new Date().getTime()

        if(!pervious && options.leading === false) pervious = now
        var remaining = wait - (now - pervious)
        if(remaining <= 0 || remaining > wait){
            if(t){
                clearTimeout(t)
                t = null
            }
            pervious = now
            fn.apply(self,args)
            if(t) self = args = null
        }else if(!t && options.trailing !== false){
            t = setTimeout(function(){
                pervious = options.leading === false ? 0 : new Date().getTime()
                t = null
                fn.apply(self,args)
                if(t) self = args = null
            }, wait);
        }
    }
    throttle.cancel = function(){
        clearTimeout(t)
        pervious = 0
        t = null
    }
    return throttle
}

到这里，我们基本就实现了防抖。
