js创建对象
js
js 2019-01-08

前言

程序员找不到对象怎么办，哼，我们会自己 new 一个对象出来。扎心了。不过我们创建的对象有好也有坏，本文我们来捋一捋。

字面量方式

我想要个女朋友，那创建一个呗

var Amy = {
  name: 'Amy',
  age: 18
}
var Wendy = {
  name: 'Wendy',
  age: 18
}

简单直接，但是要广撒网，创建多个对象，那不是很麻烦，复制粘贴，那不是成本很高，重复代码太多了，那优化一下

工厂模式

要知道我们是生成对象，所以我们来做个加工厂。

function createGirl(name,age,height){
  var o = new Object()
  o.name = name
  o.age = age
  o.height = height
  o.getAge = function(){
    console.log(this.age)
  }
  return o
}

var Amy = createGirl('Amy',18,155)
var Wendy = createGirl('Wendy',18,165)
var Anna = createGirl('Anna',18,175)

直观看到重复代码减少了，但是她们都是 Object 类型,这不太好吧，不行，换一种方式

构造函数模式

function CreateGirl(name,age,height){
  this.name = name
  this.age = age
  this.height = height
  this.getAge = function(){
    console.log(this.age)
  }
}

var Amy = new CreateGirl('Amy',18,155)
var Wendy = new CreateGirl('Wendy',18,165)
var Anna = new CreateGirl('Anna',18,175)

这样她们就有类型的概念了，但是每创建一个实例，每个方法都要被创建一次，也是就说我本想对每个对象说我有一套房子，没想到创建她们要三套房子。成本太高了

Amy.getAge === Wendy.getAge // false

原型模式

function CreateGirl(){}
CreateGirl.prototype.age = 18
CreateGirl.prototype.getAge = function(){
  console.log(this.age)
}

var Amy = new CreateGirl()
var Wendy = new CreateGirl()
Amy.getAge() //18
Wendy.getAge() // 18
CreateGirl.prototype.age = 19
Amy.getAge() //19
Wendy.getAge() // 19

方法虽然不会重新创建，但是所有的属性和方法都共享了，还不能初始化参数，这两对象不就一模一样了么。这样代码写的很繁琐，我们可以精简一些

原型字面量对象

function CreateGirl(){}
CreateGirl.prototype = {
  constructor: CreateGirl, //通过constructor属性找到所属构造函数
  age: 18,
  getAge: function () {
    console.log(this.age);
  }
}
var Amy = new CreateGirl()
var Wendy = new CreateGirl()

就是原型模式的字面量写法，该怎样还是怎样。那我们把构造函数和原型模式结合起来是不就能取长补短了

组合模式

//构造函数
function CreateGirl(name,age,height){
  this.name = name
  this.age = age
  this.height = height
}

//原型
CreateGirl.prototype = {
  constructor: CreateGirl, //通过constructor属性找到所属构造函数
  getAge: function () {
    console.log(this.age);
  }
}

var Amy = new CreateGirl('Amy',18,155)
var Wendy = new CreateGirl('Wendy',18,165)
var Anna = new CreateGirl('Anna',18,175)

环肥燕瘦，该私有的私有，该共享的共享，但是如果想要更好的封装性呢

动态原型模式

function CreateGirl(name,age,height){
  this.name = name
  this.age = age
  this.height = height
  if(typeof this.getAge != "function"){
    CreateGirl.prototype.getAge = function () {
      console.log(this.age)
    }
  }
}

var Amy = new CreateGirl('Amy',18,155)

确实好看不少，但是要注意一点不要在函数内部使用原型字面量方式重写原型，因为这种写法是先创建的实例，然后在修改的原型，要是用原型字面量重写原型，会切断现有实例和新原型之间的联系，导致方法实例上无此方法

寄生构造函数

function CreateGirl(name,age,height){
  var o = new Object()
  o.name = name
  o.age = age
  o.height = height
  o.getAge = function(){
    console.log(this.age)
  }
  return o
}
CreateGirl.prototype.getName = function(){
  console.log(this.name)
}

var Amy = new CreateGirl('Amy',18,155)
Amy.getAge() // 18
Amy.getName() // Uncaught TypeError: Amy.getName is not a function
console.log(Amy instanceof Person) // false
console.log(Amy instanceof Object)  // true

和工厂模式类似，只是用了new 初始化实例，同样都无法指向构造函数，所以和构造函数的原型没有任何关系

稳妥构造函数模式

function createGirl(age){
  var o = new Object()
  o.getAge = function(){
    console.log(age)
  }
  return o
}
var Amy = createGirl(18)
Amy.getAge() // 18
Amy.age = 20
Amy.getAge() // 18
console.log(Amy.age) // 20

也就是说，没有公共属性，并且其方法中不引用this的对象，也无法指向构造函数

总结

各有优劣结合实际业务情况选择创建对象方式，关于 es6 的 class 涉及方面较多，慢慢理清楚再开篇
